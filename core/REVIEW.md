## Cold review — the reviewer payload contract
**Cold review (T1+).** A fresh agent gets **the change itself** + the spec **as a contract to falsify** (never the case for *why* it's right) + invariants + **claimed tier** — **never the author's advocacy** ('what we verified', 'gates passed', 'N cases pinned') and never the build conversation. The question is *"where does the code violate the contract — and where is the contract itself (any backstop it names) wrong or unprovable?"*, not *"is the design's argument sound?"* The **contract** = the design's **mechanism** (what it does) + acceptance criteria + invariants + the **mitigation-claims table** — *not* its **rationale** ("why it's safe"), which is **redacted from the free-adversary pass** (released only to a later folded pass). Every **semantic rule/definition** in the contract carries its own **provenance** (authoritative source · forbidden counterexamples · raw-truth fixtures): **acceptance criteria cannot define away an invariant**, and the **free adversary must judge whether the contract itself is sourced and true** — a wrong contract faithfully implemented still ships the bug (no machine catches this; it is judgment). Mandate: *try to break it; **accept nothing as true without verifying it against the code/source** — every claimed mitigation, backstop, and contract premise is a hypothesis you confirm or refute, never a fact you inherit; default FAIL on uncertainty; flag if it looks higher-risk than its claimed tier.* Verdict = per-dimension PASS/FAIL + evidence + the exact line; **any FAIL ⇒ overall NO-GO.** Dimensions: correctness · provenance-exactness · reversibility/blast-radius · failure-mode safety · **deploy-order safety** (code — *including the mechanical gate itself* — must not assume a migration / column / flag created by a step that runs *after* the auto-deploy; guard or flag-gate it) · **contract/additive safety** (a tool's output shape stays additive — never break a connector) · **IO** (when a deterministic/cheaper layer feeds a higher-intelligence consumer: it must surface/verify, not decide-for or limit) · **mitigation integrity** (every "safe because Y" claim has a *risk-isomorphic* canary + a backstop-disabled control — see the invariant; a generic canary proves nothing; unproven ⇒ FAIL). **Panel** = **1 free-roaming adversary (owns no dimension)** + angle-assigned reviewers (≥2 T2 / ≥3 T3) covering the dimension set; verdict artifact + payload manifest (named per `core/BINDINGS.md`) for T2/T3. **Shift-left (additive):** cold-check *proposed* designs/code *before* building, and any critical work at any tier (live-path · chain/stateful · corpus-touching · held below full certainty) — feeds the mandated panel, never replaces it, never lowers a tier. Cold reviewers run at **peer tier** (never below the Builder, so a blind peer's FAIL is a legitimate flag, not a lower-layer veto; a *disputed* FAIL → frontier-PM adjudication per Gate states) and do **not** consume the external gate.

## Decorrelation — where coverage actually comes from
**Decorrelation — where coverage actually comes from.** Reviewer *count* and model *family* add **no coverage if they share the framing input**: correlated framing yields confident-wrong clears (a mitigation seven reviewers believed and none ran). Decorrelate in priority order — **(1) posture [highest]:** ≥1 **free-roaming adversary** whose payload is *exactly* the code + the **redacted contract** (mechanics + criteria + invariants + mitigation-claims table) — **no prior findings, no verdicts, no rationale** — told *"break it any way; the contract's claims are hypotheses to falsify, including any backstop it names."* No assigned angle. **(2) model family:** a different-family lens — counts only if its input is *also* unframed (run the cross-family tool **without** the folded-findings payload for this pass). **(3) angle [coverage]:** angle-assigned reviewers own distinct hunt-classes (correctness · concurrency · provenance/contract · failure-mode · security · deploy-order · **mitigation-integrity**), each unframed. **Two passes:** a free/blind pass (no prior findings) **then** a folded-delta pass (may receive prior findings, to hunt the gaps). The free adversary is **additive and owns no dimension**; the angle-assigned reviewers (≥2 T2 / ≥3 T3) cover the dimensions between them. **Enforcement — Owner spot-check, not machinery.** The verdict **records what each reviewer was given** {role · family · files · pass-type} and which pass was the **free/unframed** one; a T2/T3/chain verdict is **NO-GO** unless a free pass is named. Because reviewers self-report posture, the **Owner spot-checks the free pass** (was it really given no prior findings/advocacy?) and the mitigation verdicts — that human check is the verifier, not a hash. A **single-reviewer gate** (T1 or a minor core-doc clause) defaults to the free-roaming posture + a mandatory mitigation-integrity check. **Pass-types** are canonical + manifest-recorded: `free` forbids prior-findings/rationale/folded text; `folded` allows them. The **free adversary owns no dimension but may raise a FAIL on any**; that FAIL stops the reviewer pass and must appear in the PM disposition record, but it does not predetermine the disposition. *Foggy-multi-angle is the failure; clear-vision-first is the fix.*

## Cross-family lens — review the ARTIFACT, not just a diff
**Cross-family lens — review the ARTIFACT, not just a diff.** A different model **family** reviews the change, run in **two passes** (per Decorrelation): a **free/blind pass** fed only the redacted contract + invariants (**no** prior findings — for posture-decorrelation), then optionally a **folded pass** fed the **prior same-family findings** to hunt the gaps the panel left. Either pass uses one of two artifact-modes: (a) **DESIGN gate** — the **design/spec doc itself is the artifact**; (b) **CODE gate** — the **diff + the design/acceptance context + the changed code files** (each up to a ~120KB per-file cap; larger files fall back to diff hunks, untracked only under ~300KB). Read-only; never touches the live service. Value = **decorrelated blind spots**. **Lesson (load-bearing):** at a *design* gate — or on a docs-only changeset — a diff-only lens at most grades a tracked doc diff (the wrong lens) and finds **nothing** useful ("docs only, SAFE"); feeding the design as the artifact has caught real issues a same-family panel missed. That diff-only miss is **payload starvation, not the model** — at a design gate, feed the design. Fires with the multi-agent panel (T2/T3 + chain/stateful) when the Owner confirms availability (asked once per thread). **At T2 it is skipped cleanly when unavailable; at T3 it is REQUIRED** (`core/WORKFLOW.md` § Steer tier table — T3 owes BOTH cross-family families), so an unavailable lens at T3 takes the substitution rule below, and if no different family can be seated the artifact is **not gradable at T3** → escalate to the Owner rather than passing a reduced verdict as full. **Additive** (never replaces a mandated reviewer or the panel count), **advisory** (findings → PM triage → approved fixes re-verified; the Owner GO is final), **budget-free** (does not consume the external gate). Brackets the external gate (capstone + fix-delta bookend, per the final-gate rule). **Tradeoff** vs a same-family cold *agent*: more independent (different weights) but reasons from the supplied artifact+context; a same-family agent has live full-repo depth but shares the author's blind spots — use both when a fix is subtle. *(The runner + access are a project tool — see `core/BINDINGS.md`.)*

## External gate — the authority pass
**External gate (T2/T3, *after* cold passes).** Reviews the **code/diff + the design as a contract** — **never an advocacy ticket as the primary artifact** (a ticket of "what passed / cases pinned" anchors a false clear; the code is the artifact, the design is the contract). Reserved for this pass (conserve its budget). *(Tooling in `core/BINDINGS.md`.)* **Owner-authorized substitution (credits):** when the Owner is out of credits on a non-Claude reviewer (cross-family lens or external gate), the gate **does not drop that role — it substitutes the best available model and replicates the missing reviewer's scope/intent** (unframed, same hunt). Priority: **another *different* family first** (preserves decorrelation — Codex out → Gemini runs the cross-family role, and vice versa); only if *all* non-Claude families are out, a **same-family free-roaming Opus pass** that replicates the role (posture preserved, decorrelation reduced). The verdict **records the substitution + the decorrelation level** (full / one-family / same-family-only) — never counted as full when reduced. The **Builder-family cold panel is never substituted away** (free, always available — the spine): that is the **Claude** panel by default, and the swapped-in Builder's family under a compute-weather flip — the spine is family-relative; the shape (a fresh same-family blind panel always runs) is not.

## Artifact-class review physics — code vs. instruction artifacts
*What changes per class is **what reviewers optimize** and **how findings are dispositioned** — the ladder's spine (cold-first · decorrelation · Owner GO) is unchanged. Classify each **file** in the changeset, not the repo: **CODE** = executed by machines (`src/`, `pil/`, scripts, schemas, config, CI). **INSTRUCTION** = executed by LLMs (doctrine, prompts, skills, templates, CLAUDE.md, this framework). A mixed changeset gates each file under its own class; **tie-break by what executes the changed text** — an LLM-run block inside a code file (an embedded review prompt, a tool-description string) takes instruction physics.*

**Code physics: the sections above, unchanged — a *reachable* finding folds by default, but `REMEDIATE` requires a written `TRIGGER` (the reachable path in the declared threat model — `core/WORKFLOW.md` § Gate); a finding with no trigger is `DEFER`/`DECLINE`. Plausibility never folds a code finding any more than an instruction one — the difference is only the default (code: fold the reachable; instruction: decline-or-reference).**

**Instruction physics (the deltas):** an instruction artifact is a program run by a nondeterministic interpreter whose attention degrades with length — **length is a first-order cost, and a reviewer's "an LLM could misread this" is a hypothesis, not a verified failure.**
1. **Budget is a gate.** Every instruction artifact declares a word/token budget (standing record: `core/REPO_INVARIANTS.md` § Instruction-artifact word budgets, plus the mechanical byte caps in `scripts/check-doc-size.mjs`; **undeclared ⇒ the gate's first act is declaring one with the Owner**). Past budget, folding is zero-sum — a fold must displace text or land in a reference layer *(= a linked doc outside the executor-loaded core)*. New precision lands in the reference layer **by default**; the executor-facing core changes only for a new/changed **gate, emit, or stop**.
2. **Findings carry a price.** Reviewer briefs require severity × likelihood **and the token cost of the proposed fix**, weighed against what it displaces. Disposition default flips to **decline-or-reference** unless the finding changes a gate/emit/stop (code default: fold).
3. **Reproduce before folding (BLOCKER/HIGH).** A claimed misreading is **demonstrated on a fresh executor** (the text + the scenario — does it actually misread?) before the fix is folded — the instruction analog of "reproduce the bug." Plausibility alone never folds a HIGH.
4. **A standing subtractive leg.** Every instruction ladder includes one reviewer briefed **only to CUT** (duplicate statements · rationale-as-rule · dead weight · register inflation); **at T1 the single pass carries the CUT brief in addition to its default free-roaming + mitigation-integrity posture — never instead of it.**
5. **Execution evidence outranks review opinion.** Comprehension sims, production-format probes, pilots, and live-run compliance markers (ACK/EMIT) are the tests: a pilot **closes the findings its scenarios exercised** and **reopens** what review passed but the run failed. *(Evidence informs PM disposition; it skips no mandated gate.)*
6. **Tier by blast-radius × reversibility per Steer, never by reflex:** wording tweak that changes NO gate/emit/stop semantics = **T1** (self-check + one cold pass) · new/changed gate/emit/stop = **T2** (cold + x-family) · controlling-doc restructure = **T3** (full ladder — **core docs keep the core-doc gate SHAPE: no external gate for PROSE; embedded executable content — schemas, commands — follows the code tiers per Steer's exception; this rule sets depth, not shape**). **On a controlling doc, unsure-whether-semantic ⇒ T2 (fail closed). An instruction file that DIRECTS PROD WRITES (the Runbook) tiers by the write it directs — never as "just docs."** The full ladder is never the default for instruction work.
